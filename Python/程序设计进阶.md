		==**良好的程序设计风格包括源程序文档化；且清晰第一，效率第二**==
	软件设计
		软件详细设计的阶段任务：为软件结构图中的每一个模块，确定出来算法和局部数据结构，并用某种选定的表达工具表示算法和数据结构的细节
	结构化程序设计
		原则
			自顶向下、逐步求精、模块化、限制使用goto
		强调程序的易读性
		基本结构
			顺序结构、选择（又称分支）结构和重复（又称循环）结构
	面向对象的程序设计
		面向对象的设计理念是相对于面向过程的设计而诞生的
			==**把要实现的事情拆分成一个个步骤依次完成**==，这就是面向过程
			例如设计一个ATM机程序，要实现存入50和取出100的功能
			![[Pasted image 20260225150621.png]]
			实现思路如上，你会发现如果我还要继续加多一些属性，你就需要在函数里面装更多的参数，但是写得越多，代码就越冗余
		那么面对对象具体来说是怎么个面对对象呢？
			举回ATM的例子，ATM取钱，至少要面对ATM设备自身、账户、交易这三个对象
			对于ATM设备来说，可能有设备编号、所属银行、所在支行这三个属性
			那么我们可以构造一个“ATM类”
				![[Pasted image 20260225151812.png]]
				==**通过类来创建对象，类是对象的模板，对象是类的实例**==
				![[Pasted image 20260225152152.png]]
				那么在构造函数的时候，我们就直接用对象作为参数
				面向对象的好处是让程序逻辑更清晰，尽管代码量相对有些许增加
			同时对于方法来说，在引用方法上面也有一定的创新
				![[Pasted image 20260225152539.png]]
				如上图，面向过程的方法引用就需要这么多步骤
				但是，如下图所示，面向对象的话，我们==**将对象的属性理解为类的变量，将对象的方法理解为类的函数**==
				![[Pasted image 20260225153238.png]]
				同时，在赋予对象属性的时候我们就不用在括号写一堆参数了，直接写在类里面，最后通过对象=类，之间用对象.类里面的函数直接引用即可
		四大属性
			封装
				指的是写类的编写者可以隐藏类里面的细节，让使用者只能通过外部接口访问，进而使用类里面的东西
				外部接口可以理解为提供使用类的方法
				*就像现实中我们使用洗衣机一样，我们不需要知道洗衣机的原理就可以直接通过洗衣机上面的按钮来使用*
			继承
				类与类之间有从属关系

			抽象
				面向对象在对象的基础上抽象其属性、方法
			简单理解就是：将数据和操作结合起来/把数据和操作放在一个统一体里面 是 封装
			对象由标识、属性、方法(操作)三部分组成
