整数类型
	与数学中整数的概念一致
	可正可负，没有取值范围的限制
	我们可以用pow(x,y)函数
		计算x^y 
		当然你可以：pow(2,pow(2,15))
	整数有四种表示方式
		十进制
		二进制
			以0b或0B开头：0b010，-0B101
			b可以是大写也可以是小写
		八进制
			以0o或0O开头：0o123，-0O456
			同样的o不区分大小写
		十六进制
			以0x或0X开头：0x9a
浮点数类型
	==**和数学中的实数概念一致**==
	指的是带有小数点以及小数的数字
	浮点数取值范围和小数精度都存在限制，但在常规计算中可以忽略
		因为浮点数取值范围在:[-10^{308},10^{308}]
		精度可以达到10^{-16}
	浮点数的不确定尾数问题
		计算0.1+0.3的时候=0.4没问题
		到计算0.1+0.2的时候，=0.300000004
		这就是不确定尾数问题
		但这并不是一种bug，反而不仅仅在python中存在
		这反映了计算机是怎么实现浮点数计算的
		例如0.1计算机其实是写成二进制的：
			0.1<=>0.0001100110011....
			一共53位，大概能表示10^{-16}量级
		又因为计算机的二进制和十进制并不存在严格的对等关系，那么在输出的时候，只输出16位小数，再往后的小数没有再输出
		进而二进制表示小数只是无限接近于小数而不是完全在数学上等同
		那么怎么解决这个问题呢？
			round函数
			例如当我要比较0.1+0.2和0.3的关系，用 round(0.1+0.2,1) == 3来比较
			round(x,d),对x四舍五入，d是小数截取位数
	浮点数的科学计数法表示
		使用字母e或E作为幂的符号，以10为基数，格式如下:
		 数字 e 数字《=》数字* 10^{数字}
		 例如：4.3e-3 值为0.0043;9.6E5 值为960000.0
复数类型
	与数学中复数的概念一致
	i表示的是:i=(-1)^2,j=√(-1)
	复数有两种属性，z.real代表复数的实部，z.imag代表复数的虚部
	==**复数的实部和虚部都可以为0**==
	复数类型是进行空间变换尤其是复变函数领域里面比较重要的一个工具
数值运算操作符
	+-* / 和//
	其中因为除法例如10/3=3.33333....35
		因为得到的是浮点数结果，所以有不确定尾数
		同时python两个整数相除得到的是一个浮点数结果，这和其他编程语言是不一样的
		在这里python还设置了10//3这样的整数除法，10//3=3
	==**浮点数除法“/”，返回浮点结果。整数除法“//”**，返回整数结果==
	还有一些别的有用的运算符号
		+x=x本身
		-y=y的负值
		x%y取余运算，例如10%3 =1
		x** y幂运算，x的y次幂
			注意：当y为小数时，为开方运算，例如10** 0.5= √10
	增强赋值操作符
		x op=y <=>x=x op y
		例如x +=y <=> x=x+y
		类似地还有：-=,/=,* =,** =,%=
	例如
		3+5%6* 2//8的值
		5取余6得5，10//8直接取1，最后得到4
数字类型之间的混合运算
	不同数字类型之间的混合运算，会得到最宽的数据类型
	复数是最宽的数字类型，浮点数次之，整数最窄
	==**例如123+4.0=127.0(最终得到的是浮点数类型)**==

内置的数值运算函数
	abs(x)求解x的绝对值
	divmod(x,y)，对x,y进行商、取余数的操作,从而同时输出商和余数(x//y,x%y)
		例如divmod(10,3)返回(3,1)
	pow(x,y[,z])
		第三个参数是在幂运算之后%z
		==**在函数表示中如果一个参数是可以省略的，我们就加上一个[]**==
	round(x[,d])
		取整函数（四舍五入），其中d是保留多少位小数，==**默认=0**==
	max(x1,x2,...,xn)
		返回最大值，n不限
	min(x1,x2,...,xn)
		返回最小值，n不限

内置的数字类型转换函数
	int()
		将x变成整数类型
		数字类型转化函数的转化是不包含四舍五入的，要和round()区分
		如果输入的是字符串类型，那么int函数就会将字符串直接变成整数
		如果字符串是浮点数，那么会报错，例如ValueError: invalid literal for int() with base 10: '12.97'
	float()
		将x变成浮点数类型
		如果输入的是字符串类型，那么float()函数会把字符串变成浮点数
	complex()
		是将整数变成一个复数，其实就是增加一个虚部部分
		complex(4) = 4+0j

	更进一步，如果我工作日进步1%,休息日退步1%呢?

		我们需要循环、条件函数来解决这个问题，遇到工作日我+1%,遇到休息日我-1%
		for 变量 in range(循环的次数)
		for i in range(5):
		print(i)

		range(M,N)就是输出从M到N-1的数字
		核心代码：dayup = 1.0

	再进一步，工作日模式需要努力到什么水平，才能与每天努力1%一样呢？
		我们假设ab两个方案，
		A的方案是一年365天，每天进步1%
		B的方案一年365天，每周工作五天，休息两天，休息日下降1%。那么b需要在工作日努力几个百分点才能和a方案一样等效
		这时候我们可以用计算机一个个数值尝试
			def while 语句
		自定义一个函数
			在python里面，我们可以自定义一个自己的函数
			使用def 函数名(参数):
			例如这里，def dayUP(df):
				dayup=1
			(这里的df是参数占位符)
			之后写for i in range(365):  
			if i % 7 in [5, 6]:  
				dayup = dayup * 0.99  
			else:  
				dayup = dayup * 1.01
			return dayup(返回计算出来的dayup)
			dayfactor = 0.01
			while dayUP(dayfactor) < 37.78:
				dayfactor += 0.001
			print("工作日的努力参数是:{:.3f}".format(dayfactor))
			为什么df能和dayfactor"自动对应"
				这是函数调用时的参数传递规则。
				这段代码里最关键的一句是：
					dayUP(dayfactor)
					这句话的意思是：
					把变量 dayfactor 当前的值，传给函数 dayUP，并让函数定义里的形参 df 来接收它。
				也就是说：
					def dayUP(df):
				这里的 df 只是一个形参，你可以把它理解成函数内部临时用的名字。
				而外面这一句：
				dayUP(dayfactor)
				里的 dayfactor 是实参，也就是你真正传进去的值。
				所以执行时相当于：
				df = dayfactor
				但这个赋值只发生在函数调用的那一刻，并且只在函数内部有效。
				你可以这样理解：
				你定义函数时像是在说：
				“你以后给我一个数，我在函数内部把这个数叫作 df。”
	在上面的代码中，我们看到了很多python语法元素
		判断条件循环while，次数循环for，分支if else，函数

字符串数据类型
	字符串是什么
		由0个/多个字符组成的有序字符序列
		可以由一对''/""表示
		例如:"输入带有符号的温度值"
		本质上是字符的有序序列，可以==**对其中的字符进行索引**==
		例如:"输"是"输入带有符号的温度值"是第0个字符
	字符串的表示法
		表示单行字符串:用一对''/""
		表示多行字符串:用一对'''/"""
		从中我们可发现提供多行字符串的表示方式不就是多行注释吗
		实际上，字符串也可以被用于注释
			==**在程序中出现字符串但是并没有赋给某个变量/没有进行任何操作，那么字符串在该情况下可用于注释**==
		如果我想在字符串中包含双引号和单引号
			如果==字符串要包含双引号，就要在外层用单引号==
			==反之，要包含单引号，在外层就要包含双引号==
		如果在字符串中既需要单引号也需要双引号呢
			那么我们可以用三个'
	字符串的索引与切片
		![[mmexport1773106352710.jpg]]
		==**我们用[]获取字符串中一个/多个字符**==
		索引:返回字符串中的单个字符
			<字符串>[M]
			例如"输入"[0]，返回的就是"输"。或者可以用TempStr[-1]来获取
		切片:返回字符串中的一段字符子串
			<字符串>[M:N]
			"请输入带有符号"[1:3]或用TempStr也可以
			==**M和N可以省略，M省略时表示至开头，N省略时表示至结尾**==
		步长切片
			<字符串>[M:N:K]
			根据步长切片
			M和N可以省略，M省略时表示至开头，N省略时表示至结尾
			例如"零一二三四五"[:3]
			结果是"零一二"
			可以发现，==**切片的时候左闭右开不包含结尾**==
			又例如:"零一二三四五六七八"[1:8:2]
			从"一"提取到"八"之前
			结果:"一三五七"
			"零一二三四五六七八"[::-1]
				八七六五四三二一零
				-就是从后往前
				1是步长
