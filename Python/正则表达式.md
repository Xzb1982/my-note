正则表达式一般是用两个/包裹起来的
“//g”这里的g是修饰符，表示global（全局匹配）
	其中还有m（多行匹配multiline）、i（忽略大小写case insensitive）
"/a/g"
	意思是全局匹配字母“a”
	进一步地："/at/g"
		就是全局匹配字母"at"
"."表示除了换行符之外的任意一个字符
	例如"/at./g"，就是全局匹配字母"at+任意一个字符"
	“/a.t/g”就是a+任意一个字符+t，"/.at/g"以此类推
"[]"表示一个字符集合
	例如“/at[ce]/g”就是全局匹配at+c或者at+e的字符串
	![[Pasted image 20250925201823.png]]
	又例如"/at[cs0]/g"全局匹配at+c或者at+s或at+0的字符串
	在这个集合中，可以用"-"表示范围
		例如[a-z]就是从a到z的所有字母
		同时，[A-Z]就是从A到Z的所有大写字母
		例如"/at[A-Z1-3]/g"全局匹配at+A-Z的一个大写字母或at+1-3一个数字的字符串
		![[Pasted image 20250927145708.png]]（上面的图还包含了a-z小写字母的）
	在"[]"内，"^"可以表示取反（==注意，"^"只有在[]内才表示取反，在[]外则表示匹配每一行的开头==）
		例如"/at[^A-Z]/g"全局匹配at+除了A到Z的一个大写字母的字符串
		![[Pasted image 20250927150146.png]]
预定义的字符类
	如果每次都要在[]内部写上一堆范围，那么事情会变得繁琐起来，因此在正则表达式中预设了一些字符类方便表示
		\d表示数字(digit)，=[0-9]
		例如"/at\d/g"全局匹配at+数字类的任意一个字符
		![[Pasted image 20250927151323.png]]
		\D表示非数字，=[^0-9]
		\w表示字母、数字、或下划线（word character（单词字符））
		\W则相反，就是除了字母、数字、或下划线以外的其他字符
		\s表示空白字符（space），空格和TAB都算
		那么\S就是非空白字符

位置匹配
	在[]以外，"^"用于匹配每一行的开头、"$"用于匹配每一行的结尾
	例如"/^a/gm"就是全局、多行匹配每一行的开头之后有字母a的那一行
		不过==**在vscode里面，//m是默认的**==
	又例如"/.$/gm"就是匹配每一行结尾之前有"."的地方
		![[Pasted image 20250927154546.png]]
		注意，这里的“.”其实也是正则表达式中一个特殊的字符(除了"."，还有[], * , + )
		即"."表示除了换行符以外的其他字符
		如果不想匹配到其他字符，就需要用\转义
	其中^和$可以一起使用
		例如"/^at$/gm"就是==**开头到结尾之间只有at**==
	\b表示单词的边界
		例如“/in/gm”就是全局多行匹配单词中带有"in"的
		![[Pasted image 20250929195727.png]]
		那么"/\bin/gm"就是全局多行匹配在==单词开头==中带有"in"的
		"/in\b/gm"全局多行匹配在==单词结尾==中带有"in"的
		如果要匹配独立的单词，就需要在前后都加\b
			例如"/\bin\b/gm"全局多行匹配独立的"in"
		如果要匹配单词中间带"in"的，就要用\B
			例如"/\Bin\B/gm"

量词
	现在，我们想匹配a后面带有多个t的字符串，例如attt、attttt
	那么我们此时可以在t后面来个+（表示的是==加号前面的字符重复了≥1次==）
		例如"/at+/gm"，那么就是匹配att、atttt等a后面带至少一个t的字符串
	* 表示的是==星号前面的字符重复了≥0次==
		例如"/at*/gm"就是
	?表示的是问号前面的字符重复了0到1次
	除此之外，我们还可以用{}来制定重复次数
		"/at{3}/gm"就是全局多行匹配所有的带有attt的单词
		![[Pasted image 20250929203221.png]]
		如果只是想匹配3-5个t的话，就可以写："/at{3,5}/gm"
		![[Pasted image 20250929203415.png]]
		如果把5省略掉，那么就是至少三个t出现了
		注意，正则表达式一般是贪婪匹配的，尽管我只想要3个t，如果不指定，那么就是匹配至少3个t
	非贪婪匹配
		贪婪匹配是==**满足条件后尽可能多的匹配，直到不再满足条件为止**==
			例如atttttttt，尽管我们要3个t就可以了，但是后面跟着的t也是满足条件的，那么在这个情况下也会继续匹配
		非贪婪匹配就是满足最小的匹配条件
			那么此时我们会在{}后加个"?"
			例如"/at{3,}?/gm"
			![[Pasted image 20250929204743.png]]

分组与引用
	分组
		将多个字符作为一个整体来处理
		例如我们只需要atatat、atatatat等
		那么我们可以用"(at)"来解决
		例如"/(at) * /gm"
		同样地，at也可以用{}来限定重复的次数
			例如"/(at){3}/gm"
