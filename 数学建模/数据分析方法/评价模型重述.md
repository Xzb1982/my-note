		==你可以多你觉得重要的指标拉大差距==
	==用不同因子拆分指标==
		而指标有正向/负向，区间

		设i为行，j为列
		那么==对负向指标我们可以：①用这里面的最大值分别减去自身及下列所有数据==：$$x'_{ij} = max(x_{ij})-x_{ij}$$
			那么就变成 0 17 17 0 17；这样反而就可以用越大越好来比较
		==②对整个数据列取倒数==，同时再加上最大值防止出现0，同时加常数项k
			$$x'_{ij} = \frac{1}{k+max|x_{j}|+x_{ij}}$$
		==③在①的基础上，除以最大-最小，得到一个比例==
			这样不仅能将负向指标转正，还可以显示一些差距：$$x'_{ij} = \frac{max(x_{j})-x_{ij}}{max(x_{ij})-min(x_{ij})}$$![[Pasted image 20260113213342.png]]

那么做好这些了，我们就可以开始评价了
	![[Pasted image 20260114110143.png]]
	我们将每个指标横向线性求和，得到总评分
	但是这又带来一个问题
		每个指标的权重是怎么样的？
		如果直接求和，那么我们就是平均权重
		那么怎么求权重呢？
			①层次分析法AHP（偏主观）
			偏数值、偏议题相关的会经常用到
			![[Pasted image 20260114111057.png]]
			层次分析法通过确定权重矩阵来确定权重
			而确定权重矩阵的方式就是一一比较，例如外卖气质和作品有相似的重要性，就赋值为1
			同时，权重矩阵是上三角/下三角矩阵，也就是确定一个三角就够了
			例如黑料和演技相比那个更重要
			得到权重矩阵之后，先计算特征向量：有最大特征值法和几何平均值法这两种计算方法
			之后进行一致性检验，检验赋权重的时候有没有让矩阵不对称的地方
			![[Pasted image 20260114111700.png]]
			②熵权法
				![[Pasted image 20260114112316.png]]
				我们也可以根据数据的分布情况来计算权重
				$$\begin{align*}
对于一个数据矩阵 R = (r_{ij})_{m \times n} ;对某个指标 r_{j},有一信息熵E_{j},两者存在:\\
E_{j} = - \frac{1}{\ln m} \sum^m_{i=1} p_{ij}\ln p_{ij}
\end{align*}$$
				其中，p_ij的意思是：$$ p_{ij} = \frac{r_{ij}}{\sum^n_{j=1} r_{ij}}$$
				即==**这个指标值在整一列的占比如何，从而等价于这个指标值得出来的概率**==
				对于信息熵来说，==**熵越大信息分布越均匀，那么权重应该小==**
				**==而熵越小，信息分布越不均匀，权重应该大以拉平差距**==
				权重计算：$$ \omega_{ij} = \frac{1- E_{ij}}{\sum^n_{j=1}(1- E_{ij})}$$
				![[Pasted image 20260114114013.png]]
				![[Pasted image 20260114114029.png]]
			③TOPSIS
				通过“距离”计算评价效果的方法
				![[Pasted image 20260114114157.png]]
				和之前的AHP和熵权法不同，TOPSIS可以直接用来评价各个对象，也可以用来赋权重
				![[Pasted image 20260114114341.png]]
				在这个例子中，我们将AHP和熵权法组合，基于熵权法，我们将权重乘进去
				现在，我们先计算每一列的最大值、最小值:
				$$\begin{align*}
将最大值设为 r^+_{j},最小值设为r^-_{j}\\
之后计算每一个对象对最大值、最小值的距离：\\
第i个对象对最大值的距离d^+_{i} = \sqrt{ \sum^n_{j=1}(r^+_{j}-r_{ij})^2 }\\
第i个对象对最小值的距离d^-_{i} = \sqrt{ \sum^n_{j=1}(r^-_{j}-r_{ij})^2 }
\end{align*}$$
				计算评分：$$Score_{i} = \frac{d^-_{ij}}{d^-_{ij}+d^+_{ij}}$$
				一般是AHP+TOPSIS/熵权法+TOPSIS
			④CRITIC多权重组合
				CRITIC评价：
				![[Pasted image 20260114115402.png]]
				一方面先计算对比性和相关性（其实就是矛盾性，但我认为其本质意思是是否相关）i为行，j为列
				对比性用来衡量“能不能拉开差距”：$$\sigma_{j} = \sqrt{ \frac{\sum^m_{i=1}(x'_{ij})- \bar{x'_{j}}}{m-1} }$$
				矛盾性用来衡量“是不是在提供独立信息”：$$f_{j} = \sum^m_{i=1} (1- r_{ij})$$
				![[Pasted image 20260114120846.png]]
				之后计算其信息承载量：$$\begin{align*}
C_{j} = \sigma_{j}f_{j} (差距\times 能否提供独立信息) \\
\omega_{j} = \frac{C_{j}}{\sum^n_{j=1}C_{j}}\\
计算得分： S_{i} = \sum^n_{j=1} \omega_{j}x'_{ij}
\end{align*}$$
				这是CRITIC法来计算得分的方式
				那么怎么用CRITIC法综合AHP和熵权法的权重呢？
					先让 AHP 和熵权法各自给出一套指标权重向量，然后用 CRITIC 判断两套权重里哪一套“信息量更大、与另一套更不重复”，最后按方法权重把两套权重加权融合成一套最终权重。
					设有n个指标：$$\begin{align*}
AHP中你会得到\omega^{(A)} = (\omega_{1}^{(A)},\omega_{2}^{(A)},\dots,\omega_{n}^{(A)})^{T},在熵权法会得到\omega^{(E)} = (\omega_{1}^{(E)},\omega_{2}^{(E)},\dots,\omega_{n}^{(E)})^{T} \\
满足 \omega_{j}\geq 0, \sum^n_{j=1} \omega_{j}=1
\end{align*}$$
					第一步，把“多套权重”拼成一个矩阵（这一步是关键的建模视角转换）。  
					把“指标”当作行，把“方法”当作列：$$\begin{bmatrix}
\omega^{(A)}_{1},\omega^{(E)}_{1} \\
\omega^{(A)}_{2},\omega^{(E)}_{2} \\
\dots ,\dots \\
\omega^{(A)}_{n},\omega^{(E)}_{n}
\end{bmatrix}$$
					这里每一列是一种方法给出的“指标重要性分配”，你接下来要用 CRITIC 给这两列分配方法权重
					你可以先无量纲化，之后用 CRITIC 计算“方法层”的对比性与矛盾性（==根据i为行，j为列，那么此时的j就是不同方法==），最后得到方法的信息量与方法权重。第五步，线性融合得到最终指标权重。用最终权重去算综合得分或做 TOPSIS 等评价。
				![[Pasted image 20260114121130.png]]
				![[Pasted image 20260114121247.png]]
			⑥随机权重法（写灵敏度分析才会用）
				![[Pasted image 20260114123044.png]]
				![[Pasted image 20260114123230.png]]
				以随机TOPSIS为例，我们先以TOPSIS法算出评价矩阵
				第一步，你先确定一个基准权重向量omega,这个omega可以是来自 AHP、熵权法、CRITIC、等权，或者你前面已经合成好的权重。关键约束是：$$\begin{align*}
\omega_{j}\geq 0, \sum^n_{j=1} \omega_{j} =1
\end{align*}$$
				第二是规定“随机权重”的生成机制。  
				生成机制必须保证权重仍然满足非负且和为 1。有两种方案，1是相对扰动后再归一化，2是Dirichlet 抽样
				第三步，把每一组随机权重代入 TOPSIS，得到一组得分。
				最后用统计量把“随机结果”变成你图里那种“均值 + 不确定区间”。

评价模型
	PCA主成分分析
		每道菜有“辣度、麻度、油量、咸度、重口评级”等指标，彼此常常一起高或一起低。
		- PCA 做什么：把这些强相关的指标“转成”两根好理解的轴。
	    - PC1≈“重口味指数”（辣/麻/油/咸一起高→分数大）。
	    - PC2≈“辣↔麻的偏好”（辣载荷为正、麻载荷为负/反之）。
	    - 局限性
		    仅适用于线性关系（一起变化）
		每月在“餐饮、酒吧、打车、外卖、娱乐、穿搭”等的花费。
		    - PC1≈“社交/外出消费强度”（餐饮/打车/娱乐一起高）。
		    - PC2≈“固定 vs 可选开支”（房租/水电与娱乐/网购呈相反载荷）。
			- 用途：把十几个科目压成两三根“预算主轴”，一眼看出你是哪类花钱模式。
		注意：
			**PCA找的是“方向”（主成分）而不是把变量分成小组。**  
			每个主成分都是“所有原始变量的**线性组合**”。  
			例：PC1 = 0.71·A + 0.69·B + 0.12·C。
			一个是大方向PC1，一个是跟大方向垂直的方向PC2

理想解/折中解
	TOPSIS理想解法
		在多指标空间里，把每个方案当成一个点；找出**理想最好点**（各指标最优）和**理想最差点**（各指标最劣），谁**离最好最近、离最差最远**，谁就更优。
	VIKOR
		在多指标打分时，VIKOR同时看两件事：  
			**S = 平均不满意度**（整体表现好不好）  
			**R = 最大遗憾**（最差那一项拉胯不拉胯）  
		再用一个权衡系数 **v**（0～1）把二者合成 **Q**，选 **Q 最小**的“折中解”。
		- **v 越大**：更看重“多数指标都不错”（群体满意，像平均分）。
		- **v 越小**：更看重“没有明显短板”（最弱项不能太差，像木桶短板）。
	TOPSIS和VIKOR的比较
		共同点
			- **都要把指标同向化并标准化**（消除“越低越好/量纲不同”的影响）。
			- **都构造“理想最好/最差”作为参照**：对每个指标取样本中的**分量极值**（效益型取最大、成本型取最小）。
			- **都给出一个综合指标做排序**（TOPSIS 的贴近度 C，VIKOR 的折中指数 Q）。
		不同
			**理想点的“用法”不同**
			- **TOPSIS**：在**加权归一化矩阵**上取理想最好/最差，然后算到二者的**欧氏距离**  
			- **VIKOR**：在归一化后取每列的最好  与最差 ，先把每个方案对最好点的**偏差**线性归一
			- TOPSIS 用欧氏距离，**整体几何接近**更重要。
			- VIKOR 同时看平均与“最差一项”，对**明显短板**更敏感

多指标决策（其实就是两两比较）
	ELECTRE优势关系法
		它不直接给每个方案一个分数，而是**做两两比较**。  
		对“方案 a 是否优于 b（a outranks b）”同时看两件事：
			1. **赞同度（Concordance）**：有**多少权重**的指标支持 a ≥ b（多数赞成）。
			2. **反对度/否决（Discordance / Veto）**：是否存在**某个指标特别差**到“**一票否决**”。
			只有当“赞同度 ≥ 门槛 c*”且“没有触发任何否决阈值 v_j”时，才认为 **a 优于 b**。
		# **租房三选一**（带“底线”）
			指标（权重）：房租↓(0.4)、通勤时间↓(0.3)、采光↑(0.2)、噪声↓(0.1)  
			设定**门槛**：赞同度 c* = 0.6；否决阈值：通勤**差>20 分钟**或噪声**差>8 dB**则一票否决。
			|房源|房租(元)|通勤(分钟)|采光(1–10)|噪声(dB)|
			|---|---|---|---|---|
			|A|2800|35|7|65|
			|B|3000|28|8|58|
			|C|2600|55|6|70|
			**两两比较：**
				- **B vs A**：B在“通勤(0.3)+采光(0.2)+噪声(0.1)”占优，赞同度 = **0.6**；且无否决 → **B 优于 A**。
				- **A vs B**：A只在“房租(0.4)”占优，赞同度 0.4 < 0.6 → 不成立。
				- **A vs C**：A在“通勤(0.3)+采光(0.2)+噪声(0.1)”占优，赞同度 **0.6**；无否决 → **A 优于 C**。
				- **C vs B**：C只在“房租(0.4)”占优，且通勤差 **27 分钟 > 20** 触发**否决** → **不允许 C 优于 B**。
				**优势网络：** B → A → C。  
				**排序：** **B 最优，其次 A，最后 C**。  
				直觉：即便某房源“多数项一般但没有大短板”（A），也可能凭“不过线”赢过“有致命短板”的（C）；而 **ELECTRE**能把这种“**底线/红线**”体现出来。
		ELECTRE非常适合有“**红线/底线**”的场景（如安全、合规、极端差不可接受）。
	PROMETHEE
		把它想成**一场联赛投票**：每个方案彼此“比赛”一场；在每个指标上，根据“好多少/差多少”给出一张**偏好票**（0~1 分）；再按权重把各指标的票加权汇总。
			- **正向流 φ+**：我打别人时累计拿到的票（我“压过别人”的程度）。
			- **负向流 φ−**：别人打我时别人拿到的票（我“被压过”的程度）。
			- **净流 φ = φ+ − φ−**（PROMETHEE II）：就像“净胜球”，越大越好，用来给出总排名。
			- PROMETHEE I 还会单看 φ+ 与 φ− 给出**部分排序**（可能出现“不可比”）。
		- **适合**：想把“差一点无所谓 / 差很多才算优势”的**偏好逻辑**显式写进模型；指标量纲不同、不想简单标准化时。
		- **注意**：结果**对权重与阈值敏感**，应做**敏感性分析**（调 q、p、权重看排序稳定性）。
		- **输出**：PROMETHEE I（部分排序，可出现不可比）；PROMETHEE II（净流给出完全排序）；还可用 **GAIA 图**做可视化。
	以上的ELECTRE和PROMETHEE很适合“**有容差、有红线、差多少才算数**”的场景
		例如1. **外卖/快递时效**
			- q：晚/早到 ≤ 5 分钟当没差
			- p：≥ 25 分钟才算明显更快/更慢
			- veto：超 60 分钟且无解释 → 直接否掉该商家（ELECTRE）

线性规划评价模型
	DEA数据包络分析
		**一句话**：**DEA 就是一组线性规划。**  
		对每个被评单位（DMU），DEA都会解一小个线性规划来给它“最有利但对所有人都公平”的效率分数。
			公司有 3 辆快递车，想看谁更“省”。
				- **投入**：油耗（L）越少越好
				- **产出**：行驶里程（km）越多越好

| 车   | 油耗 x（L） | 里程 y（km） | y/x（km/L） |
| --- | ------- | -------- | --------- |
| A   | 10      | 150      | 15        |
| B   | 10      | 160      | **16**    |
| C   | 12      | 168      | 14        |
				在“1 入 1 出”时，DEA 的线性规划会退化成：**效率 = 自己的 y/x ÷ 全体最大 y/x**。  
				所以分数是：
				- A：15/16=0.937515 / 16 = 0.937515/16=0.9375
				- B：16/16=116 / 16 = 116/16=1（在前沿，标杆）
				- C：14/16=0.87514 / 16 = 0.87514/16=0.875
				**怎么用？（投入导向）**
					- A 若想**不减里程**就“达标”，至少还能把油耗从 10L 降到 10×0.9375=9.37510\times0.9375=9.37510×0.9375=9.375 L（再配合松弛看具体怎么省）。
					- C 至少还能从 12L 降到 12×0.875=10.512\times0.875=10.512×0.875=10.5 L。  
				    这就是 DEA 给你的**量化改进目标**。


GRA灰色关联分析（像不像理想/标杆）
		**谁更像“参考样子”**。把每个方案在各指标上都拉到同一尺度后，量一量它与**参考序列**（可以是理想最优、也可以是某个标杆方案）的“相似度”，相似度越高越好。
		挑外卖店：看**价格（越低越好）**与**评分（越高越好）**，等权。  
		原始数据：A(¥95,4.5)，B(¥100,4.2)，C(¥120,4.8)。  
		把价格转极大型并 0–1 归一化后，理想参考为[1,1]。计算得到：
		- A 的关联度 ΓA=0.75
		- B 的 ΓB≈0.524
		- C 的 ΓC≈0.667
	    排序：**A > C > B**。——A 在价格是“最优形状”，评分也不差，整体最像“理想型”。


RSR秩和比（基于秩的聚合）
		**把每个指标先排名，再把名次加总并做个归一化**；名次和越小（或换成 1−RSR 越大）表示综合越优。它本质上是一个**基于秩的非参数综合评分**，很像投票里的 **Borda 计分**。
		选外卖店（越便宜越好、评分越高越好），给**价格权重 0.6**、**评分 0.4**。  
			数据：A(¥95,4.5)，B(¥100,4.8)，C(¥85,4.2)，n=3。
			- 价格（升序好）：C(1)、A(2)、B(3)
			- 评分（降序好）：B(1)、A(2)、C(3)
			加权秩和 S：
				- A：2.00.6×2+0.4×2=2.0 → RSR =2/3=0.667
				- B：0.6×3+0.4×1=2.2 → RSR =0.733
				- C：0.6×1+0.4×3=1.8 → RSR =0.600（**最好**，因为更看重价格）
		稳健、易讲清，但牺牲了“差距大小”的信息。
    


AHP层次分析法（专家评价，较为主观）（和后面的BWM有相似之处[[数据分析方法——评价权重系数选择模型]]）
	把大决策拆成层次→在每一层做“成对比较”问：A 比 B 重要几倍？→由比较矩阵算出权重→检查一致性（CR）→自上而下汇总得总分。
	步骤
		- **建层次**：目标 → 准则/子准则 → 方案。
		- **成对比较**：用 1–9 标度（1=同等；3=稍重要；5=明显；7=很强；9=极端；2/4/6/8 为中间值）。得 n×n 比较矩阵 A，满足 aij=1/aji, aii=1
		- **求权重**：取最大特征向量并归一化（或用“行几何平均”近似）。得到本层权重 w。
