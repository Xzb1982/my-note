熵权法（谁能拉开层次，谁就更重要）（用概率分布）
	把每个指标看成一位“传递事实的记者”
		- **大家都差不多**（波动小、分布均匀）→ 这位记者说的话**可预测**，**信息熵高**（=不提供新信息），**权重应低**。
		- **差异显著**（拉开层次、分布不均）→ 这位记者**能爆料**，**信息熵低**（=贡献了确定性/信息），**权重应高**。  
		这与决策树里“信息增益”的想法是同一个家族的度量。
	例子
		你想**客观地**给“挑外卖店”的两个指标分权重：**价格**（越低越好）、**配送时长**（越短越好）。
		- 过去 30 单的数据里：价格波动很大（¥18~¥60），时长几乎都在 28–34 分钟。
			- 做完标准化→概率→熵：
			    - **价格**分布很不均，熵 E价E_{\text{价}}E价​ 低 → d价d_{\text{价}}d价​ 高 → **权重大**；
			    - **时长**分布接近均匀，熵 E时E_{\text{时}}E时​ 高 → d时d_{\text{时}}d时​ 低 → **权重小**。  
	        **结论**：在你的历史数据里，“价格更能区分不同商家”，因此更值得给更高权重。这就是熵权法在做的客观加权。
	该用：你希望减少主观性，而且相信“谁能拉开层次，谁就更重要”的原则；指标规模不同、样本不算太小。
信息量权数法（离散度，例如CV、标准差、极差）
	**谁在样本里“更能拉开层次/区分对象”，谁的信息量就更大，权重就更高。**  
	这里的“信息量”通常用**离散度**来刻画（如标准差、变异系数、极差、IQR 等）。
	- 熵权用**概率分布的熵**（更像信息论度量），对“分布均匀”惩罚更强；
	- 信息量权数法直接用**离散度**（σ\sigmaσ/CV/IQR），实现更简单、直观，也更便于做稳健处理。
	- **适合**：样本量不小；想快速、客观地给指标分“话语权”；希望易解释、易复现。
	- **注意**：对异常值敏感 → 先做去极值/Winsorize；分组/跨期差别大 → **分组计算权重**；强相关会“重复计分” → 结合相关性检查或用 CRITIC。
变异系数法（CV）（用相对均值）
	- **直觉**：谁在各方案之间**起伏更大**（相对均值），谁更能“拉开差距”，就给谁更大权重。
	- **特点**：超快、尺度无关；但**不看指标间相关性**，μ≈0时不稳、对极端值敏感。
CRITIC 方法
	- **直觉**：**信息量=对比度×冲突度**：既要**起伏大**（对比度高），又要**不和别人重复**（与他指标**相关性低**）
	- **特点**：会**自动给“高度相关的一群指标降权”**，更“去冗余”；但需要相关矩阵，稍复杂，解释性不如AHP那样“口语化”。
	需要相关矩阵；样本少或异常值多时，建议用**Spearman**或稳健相关。

以上方法都在用“**越能区分样本好坏→越重要**”来分权。
- 🔎 区别在于**区分度怎么量**、以及**要不要惩罚冗余**：
    - **熵权**：用**概率分布的熵**量区分度；
    - **CV/信息量**：用**离散度**量区分度（CV 无量纲；“信息量法”是离散度的广义壳）；
    - **CRITIC**：在离散度基础上再乘**(1−相关性)**，避免“重复计权”。

BWM最好最差方法（专家偏好）
	**先挑出“最重要(Best)”和“最不重要(Worst)”两个锚点，只围绕它们做成对比较；用一个线性规划把这些比较“拟合”为一组最一致的权重。**  
	它本质是**主观偏好**（专家判断），但用优化来保证**一致性尽可能好**。
	- **列出指标集**，**选锚点**， **两组打分（1–9标度）**，**建模求权重**（线性规划，目标“最一致”）
	**优点**
		- 比 AHP 少很多比较：只需 2(n−1) 次，而不是 n(n−1)/2
		- 一致性**可度量且可优化**；在同等信息量下，权重往往更稳健。
		- 易与客观法**组合**（例如与熵权/CRITIC 融合)。
	**局限**
		- 需要专家能明确挑出 **Best/Worst**，有时并不容易。
		- 仍属**主观法**；“结合客观”的说法通常指**再与数据驱动权重做加权融合**，BWM 本身不读取数据。
		- 需要求解器（Excel 的“规划求解”、Python PuLP/linprog、R lpSolve 等）。

因子分析FA
	先把一堆相关的指标“压缩”为少数潜在因子，再用因子载荷（loading）把每个指标的“共性解释度”转成权重。
	“压缩”：把一堆相关指标合成少数“潜在因子”
		很多指标其实在测同一类“能力/构念”（比如“服务质量”“可达性”）。**因子分析**用这些指标的相关性，提炼出少数几个**公共因子**，每个因子就是一条“隐藏维度”，能解释若干指标为什么会一起波动。
	“载荷”：每个指标对因子的“相关/贡献”
		**因子载荷**（loading，记作 λ）≈ 指标 j 与因子 f 的相关系数。  
		把同一个指标在所有保留因子上的**载荷平方**加起来，就得到它的**共性解释度（communality）**
	“转权重”：共性越大，权越大
		最常见做法是把共性解释度当作“信息含量”，再归一化成权重。
		（也有变体会给不同因子按“解释方差占比”再加权，但核心思路不变：**谁被公共结构解释得多，谁更重要**。）
	“因子分析”从头到尾都是一个**线性模型**，它既把高维指标压缩到少量“潜变量”（线性降维），又把这些潜变量（及其解释度）再**线性地**合成为一个综合分值（从而给出了等价的“指标权重”）。

协同过滤CF
	来自**推荐系统**的个性化方法；根据“**别人像你**”或“**项目彼此像**”的行为相似性，**为每个用户学出一套偏好**。把“偏好”映射到指标上，就得到**个性化权重**。
	- 在做**面向大量用户的推荐/排序**，并且有**点击/评分/购买**等行为日志；希望每个用户都有一套不一样的“指标偏好”。
	- 若只是**一个专家给一组方案定权**，**不建议**用CF；用BWM/AHP/CRITIC更合适。

# 先判定

1. **要不要“个性化到每个用户”？**
    

- 要 → **CF 协同过滤**。
    
- 不要 → 继续。
    

2. **有没有必须体现的“专家/业务偏好”？**
    

- 有 → **BWM/AHP**（主观法，少比较且有一致性控制）。
    
- 没有/弱 → 继续。
    

3. **指标很多且强相关，怀疑存在“潜在维度/构念”？有足够样本（≈每指标 5–10 个样本）？**
    

- 是 → **FA 因子分析**（先降维/去相关，再由载荷/共性取权）。
    
- 否 → 继续。
    

4. **只想“客观、数据驱动”，按区分度来分权？**
    

- 是 → **“谁能拉开层次谁更重”一组方法**：熵权 / CV / 信息量 / **CRITIC**（见下微分工）。
