什么是跨周期模型
	在正式学习跨周期编写前，我们先做个比喻，方便更好的理解跨周期的编写格式：
		比如：我需要完成一项工作。但工作的其中一小部分，需要A来辅助帮我。
		也就是说，这项工作被分成2部分：我是这项工作的主要完成人，==A是辅助人==
	跨周期模型也是这个道理，模型也是分成2部分的
		如：我在==**日线上想引用30分钟中的MA数据**==。   那==**日线，相当于是主模型。30分钟，相当于辅助的指标。**==
		所以，跨周期一定需要分别建立2个模型：==**一个是主模型，用于装载日线所需要的判断，另一个是辅助的指标，用于装载30分钟所需要的MA数据**==
		==**编写时，在主模型中只需要把辅助的指标引用过来，实际使用时，加载主模型即可**==


==**IMPORT函数里面不能随随便便加空格，也就是#和IMPORT之间不允许有空格**==
	==在 WT8 里这一类“预处理指令”是按固定关键字匹配的==，关键字就是连续的 `#IMPORT`，它不是==“# 号 + 一个单词”的宽松解析==，所以 `# IMPORT` 会被当成两个东西，编译器根本认不出来这一行是 IMPORT 语句，后面就会在逗号、数字这些位置报一些看起来很怪的语法错。
		你写的时候最好同时满足三条，基本就不会再踩坑了。
		第一，`#` 必须紧贴 `IMPORT`，也就是只能写 `#IMPORT`，不能写 `# IMPORT`。
		第二，这一行尽量放在行首，`#` 前面不要有空格、Tab、全角空格。有些编辑器你以为没有，其实前面混进去了一个不可见字符，最稳妥的办法还是删掉整行后用英文输入法重敲。
		第三，末尾不能加分号，这个你前面那段说明里也写了。

#IMPORT [PERIOD,N,FORMULA] AS VAR 引用当前合约，PERIOD参数为N的周期，指标FORMULA的数据。
	看注释之前，先拆分模型的主要部分：
		 函数中各项所表达的含义：
	    ① # IMPORT [ , , ] AS ：这部分是跨周期的函数名（固定格式），是编写跨周期主模型中必须要写的；
	    ②  PERIOD ：表示我想要引用的K线周期级别，比如：我在日线上想引用30分钟，这里就编写MIN
		==N ：表示我想要引用的具体K线周期数，比如：我在日线上想引用30分钟，这里就编写30==
        ==FORMULA  ：表示引用的辅助指标的名称，比如，我在日线上想引用30分钟MA，这里就是装载30分钟所需要的MA数据的指标名称==
	    ③ VAR ：给引用的辅助的指标起一个新名字，比如，张三，李四，AA，BB 等
	    ④ #IMPORT  函数是没有分号结尾的！
	跨周期模型编写步骤
        确定被引用内容 + 编写主模型
	    第一步：建立被引用的模型，命名 AA
             MA5:=MA(C,5); //定义MA5为5周期均线
             MA10:=MA(C,10);  //定义MA10为10周期均线
      第二步：重新新建一个模型，即跨周期主模型
        # IMPORT [MIN,30,AA] AS VAR
        MA5Y:VAR.MA5;  //跨周期引用，30分钟AA指标中的MA5函数
        MA10Y:VAR.MA10;  //跨周期引用，30分钟AA指标中的MA10函数
        CLOSE>MA5Y,BPK;  //当前周期最新价高于30分钟周期的MA5时，执行买平开指令
        CLOSE<MA10Y,SPK;  //当前周期最新价低于30分钟周期的MA10时，执行卖平开指令
        AUTOFILTER;  //启用一开一平信号过滤机制![[Pasted image 20260113180716.png]]
	注：  
	1、==**PERIOD为周期，N为具体的参数，FORMULA为引用指标名，VAR为定义变量名（此变量名不能以数字开头）；**==  
	2、PERIOD支持如下周期：
		MIN（分钟周期），HOUR（小时周期），CUSHOUR（自定义小时周期），DAY（日周期），WEEK（一周），MONTH（月周期），QUARTER（一季度），YEAR（年周期）；  
	3、支持引用自定义周期；  
		如#IMPORT [MIN,2,MACD] AS VAR//引用两分钟周期MACD指标数值  
	4、N必须为≥1的整数，周、季周期，N写入>1的数，按照1计算；  
		例如：#IMPORT [WEEK,2,FORMULA] AS VAR//默认引用的是一周的指标；  
	5、==引用常规小时周期使用HOUR，引用自定义小时周期需要使用CUSHOUR。==  
	6、该函数不支持加载到量能周期使用；  
	7、该函数可以小周期引用大周期，也可以大周期引用小周期；  
	==**8、被引用的指标中不能存在引用；**==  
	9、FORMULA引用指标名可以为字母、汉字或数字命名的指标；  
	10、定义变量名不能与函数名重复；  
	==**11、一个模型中#IMPORT、#CALL、#CALL_PLUS、#CALL_OTHER总的语句个数不能超过6个；**==  
	12、==**使用该函数编写末尾不能编写分号**==。  
	例1：  
		CC:REF(C,1);//定义一个周期前的收盘价  
		保存指标，命名为AA  
		#IMPORT[DAY,1,AA] AS VAR  （==**也就是模型AA在这里面的名字变成了VAR**==）
		CC:VAR.CC;//跨周期引用昨天的收盘价  
	例2：  
		CC:C;//定义收盘价  
		保存指标，命名为CC  
		#IMPORT[DAY,1,CC] AS VAR  
		CC:=VAR.CC;//跨周期引用日周期上的收盘价  
		CC1:REF(CC,1);  
		//要引用的数据需要写在被引用的指标里，不能写在IMPORT模型中。  
		//例1中的CC指标引用日周期上前一个周期的收盘价，需要在被引用的指标中取一个周期前的收盘价，  
		例2中写在IMPORT模型中则表示取小周期上一个周期前的值  
	例3：  
		CC:=REF(C,1);//定义一个周期前的收盘价  
		保存指标，命名为AA  
		#IMPORT[CUSHOUR,6,AA]AS S  
		CC1:=S.CC;//跨周期引用自定义6小时周期的一个周期前的收盘价  
		#IMPORT[MIN,1,AA]AS R  
		CC2:=R.CC;//跨周期引用自定义1分钟周期的一个周期前的收盘价
	例4：
		要求：1 小时周期5、10、20的均线多头排列，5 分钟K指标金叉，做多；
		1 小时周期5、10、20的均线空头排列，5分钟KD指标死叉，做空；
		//Step1：编写被引用的均线指标， //保存该函数为 MM
		M1:MA(C,5) ;  M2:MA(C,10) ; M3:MA(C,20) ;
		//Step2：新建模型，命名DKMA
		//调用 1 小时 K 线周期的均线指标
		#IMPORT[CUSHOUR,1,MM] AS MM
		MA1:MM.M1;
		MA2:MM.M2;
		MA3:MM.M3 ;
			也就是说这里的周期，在MM这里面已经被拆分成为5,10,20了
			那么需要IMPORT函数在引用时对其进行激活
